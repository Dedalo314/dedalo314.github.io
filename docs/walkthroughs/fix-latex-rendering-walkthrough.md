# Walkthrough: Fixing LaTeX Double Rendering

## Summary

This walkthrough documents the fix for the double-rendering issue of LaTeX equations in the blog. The issue was caused by KaTeX outputting both HTML and MathML versions of the equations, where the MathML version was visible as plain text due to either missing CSS or browser rendering behavior.

## Steps Taken

### 1. Research and Reproduction

- Used Playwright to inspect the rendered HTML of the blog posts.
- Found that both `.katex-html` and `.katex-mathml` were present and visible.
- Confirmed that `.katex-mathml` had `display: inline`, making it visible alongside the styled version.

### 2. Configuration Change

- Updated `astro.config.mjs` to configure `rehype-katex` with `output: "html"`.
- This ensures that only the styled HTML version is generated by the markdown processor, preventing the duplication at the source.

### 3. Safety Styling

- Added a global CSS rule in `BaseLayout.astro` to explicitly hide `.katex-mathml` using `display: none !important`.
- This provides a secondary layer of protection in case any other part of the system (or future updates) outputs MathML.

### 4. Verification

- Updated `src/test/e2e/blog.test.ts` to include checks for MathML visibility.
- Added a new test case for the `relativistic-rocket` post which contains multiple complex equations.
- Ran `npx playwright test` and verified that all 4 tests passed, ensuring equations are visible, styled, and not duplicated.

## Evidence

- Playwright test results showed `MathML visible: false` after the fix.
- Visual inspection (via inspection script) confirmed that only the `.katex-html` version remains active in the DOM when `output: "html"` is used.

### 5. Integrity Fix

- Corrected the `integrity` attribute for the KaTeX CSS in `BaseLayout.astro` to match the actual file content, resolving a resource blocking error (SRI) that prevented styles from loading.
